# Tech Entrepreneur Sprint - 투자 프로그램 MVP PRD

## 1. 한 줄 요약 (One-liner)

교육생이 가상의 투자자가 되어 팀별 상세 페이지를 확인하고, 제한된 투자금(총 1,000만원)을 팀별로 나누어 실시간 단건 투자하면 시스템이 자동으로 투자 현황과 순위를 집계하는 모바일 UI 기반 웹 투자 시뮬레이션 서비스

## 2. 제품 배경 & 기획 의도

### 2.1 배경

* 본 프로그램은 단순한 발표 평가가 아니라, 문제-가치-기술을 종합적으로 판단해 자원을 배분하는 의사결정 경험을 제공하는 데 목적이 있다.
* 투자 행위는 교육생이 기술을 수단으로 사고하고 설명하는 기업가정신을 가장 직관적으로 학습할 수 있는 장치이다.

### 2.2 기획 의도

* "기술을 얼마나 잘 만들었는가"가 아닌 "왜 이 문제에 이 기술이 적합한가"를 판단하게 함
* 모든 교육생을 투자자로 참여시켜 몰입도와 참여도 극대화
* 실시간 수치와 결과를 통해 학습 피드백 제공



## 3. 목표 (Goals)

### 3.1 MVP 목표

* 투자자 1인당 총 1,000만원 투자금 자동 지급
* 팀별 10만~1,000만원, 10만원 단위 단건 투자 가능
* 실시간 남은 투자금 확인
* 팀별 투자금 자동 집계 및 순위 산출

### 3.2 성공 기준 (Success Metrics)

* 투자 참여율 95% 이상
* 운영자 수작업 없이 투자 결과 산출
* 투자 종료 후 5분 이내 결과 공유 가능



## 4. 사용자 정의 (Users)

* **투자자(교육생)**: 팀 발표를 듣고 투자 의사결정을 수행
* **관리자(Admin)**: 팀/콘텐츠/투자 상태를 관리하는 교육운영매니저



## 5. 핵심 사용자 시나리오 (Web 기반, 모바일 UI)

### 투자자(교육생)

1. 투자 사이트 접속 (모바일 UI)
2. 본인 소속 팀 선택
3. 전체 팀 리스트 확인
4. 특정 팀 선택 → 팀 상세 페이지 진입
5. 투자 금액 입력 (10만~1,000만원, 10만원 단위)
6. 투자 실행 (단건 투자)
7. 남은 투자금 실시간 확인
8. 총 투자금 1,000만원 초과 시 투자 불가
9. 다른 팀에 추가 투자 반복 (이미 투자한 팀에는 재투자 불가)



## 6. 핵심 기능 요구사항 (Core Features)

### 6.1 투자자 기능 (Must-have)

* 총 투자금 1,000만원 자동 부여
* 전체 팀 리스트 조회
* 팀별 상세 페이지 제공
* 관리자 업로드 PDF/이미지 열람
* 팀별 투자 금액 입력 및 즉시 투자
* 남은 투자 가능 금액 실시간 표시
* 한 팀당 1회만 투자 가능 (수정·재투자 불가)

### 6.2 관리자 기능 (Must-have)

* 팀 수 설정(최대 30팀)
* 팀명, 팀 설명 관리
* 팀별 상세 페이지용 PDF/이미지 업로드
* 투자 시작/종료 제어
* 팀별 총 투자금 실시간 집계
* 투자 결과 및 순위 자동 정렬



## 7. 비기능 요구사항 (Constraints)

* 웹 페이지 기반 개발 (모바일 UI 우선)
* Git 기반 정적/웹 서비스 배포
* 동시 접속자 약 30명 안정 처리
* 간단한 상태 관리로 실시간 투자 반영



## 8. 규칙 및 정책 (Rules)

* 투자자 1인당 총 투자금은 1,000만원
* 투자 단위는 10만원 단위
* 모든 투자금은 최종적으로 1,000만원을 모두 사용해야 함
* 투자는 팀별 단건 투자 방식으로 진행
* 이미 투자한 팀에는 재투자 및 투자금 수정 불가
* 투자 중 남은 금액은 실시간 표시



## 9. 리스크 & 대응

| 리스크 | 대응 |
| --- | --- |
| 중복 투자 | 팀별 1회 투자 제한 로직 |
| 동시 클릭 충돌 | 서버 단 투자 금액 검증 |
| 네트워크 이슈 | 관리자 강제 종료/결과 확정 기능 |

## 10. 추가 제안 기능 (Nice-to-have)

### 관리자 중심 기능

* 전체 팀 투자 현황 대시보드
* 모든 팀의 총 투자금, 투자자 수, 순위 한 화면에서 확인
* 발표 종료 직후 결과 공유용 화면으로 활용

### 투자자 경험 개선 기능

* 투자 완료 여부 표시
* 이미 투자한 팀은 '투자 완료' 상태로 표시
* 미투자 팀과 명확히 구분되어 의사결정 흐름 명확화

### 시각화 기능

* 팀별 투자 비율 시각화 (바 차트)
* 전체 투자금 소진 진행률 표시



## 11. 범위 제외 (Out of Scope - MVP)

* 로그인/인증 시스템
* 투자 사유 입력
* 팀 간 비교 분석 기능



## 12. 개발 일정 (Rough Timeline)

* 기획 확정: 1일
* 페이지 개발: 5~7일
* 현장 테스트 및 배포: 1일



## 13. Definition of Done (출시 기준)

* 모든 투자자가 오류 없이 단건 투자 가능
* 남은 투자금/팀별 투자금 정확히 반영
* 팀별 투자 완료 상태 명확히 표시
* 관리자가 전체 팀 투자 현황을 한 화면에서 확인 가능
* 결과 및 순위 자동 산출 및 즉시 공유 가능

---

## 화면 구조 설계 (모바일 UI 기준)

### 1. 전체 화면 구성 개요

* 모든 화면은 모바일 세로형 UI 기준
* **상단**: 프로그램명 + 남은 투자금 표시
* **하단**: 주요 행동 버튼 배치

### 2. 화면 1 - 투자자 시작 화면 (팀 선택)

* **목적**: 투자자 식별 및 진입
* **구성 요소**: 프로그램 타이틀, 본인 팀 선택 드롭다운 또는 버튼 리스트, [투자 시작하기] 버튼

### 3. 화면 2 - 전체 팀 리스트 화면

* **목적**: 모든 투자 대상 팀 확인
* **구성 요소**:
  * 팀 카드 리스트 (최대 30팀): 팀명, 한 줄 설명, 투자 상태 뱃지 (미투자/투자 완료)
  * 상단 고정 영역: 남은 투자금 표시, 전체 투자 진행률 바
* **동작**: 팀 카드 클릭 시 팀 상세 페이지 이동

### 4. 화면 3 - 팀 상세 페이지

* **목적**: 팀 정보 확인 후 투자 결정
* **구성 요소**:
  * 팀명, 팀 발표 자료 영역 (PDF/이미지 뷰어)
  * 투자 입력 영역: 투자 금액 입력 (10만원 단위), 현재 남은 투자금 표시, [이 팀에 투자하기] 버튼
* **제약 조건**: 이미 투자한 팀인 경우 투자 입력 비활성화 및 '이미 투자 완료한 팀입니다' 안내 표시

### 5. 화면 4 - 투자 완료 확인 모달

* **목적**: 실수 방지 및 최종 확정
* **구성 요소**: 투자 팀명, 투자 금액, 투자 후 남은 금액, [확인] / [취소] 버튼
* **동작**: 확인 클릭 시 투자 즉시 반영 및 팀 상태 '투자 완료'로 변경

### 6. 화면 5 - 관리자 대시보드 (PC/태블릿 권장)

* **목적**: 전체 투자 현황 관리 및 결과 공유
* **구성 요소**:
  * 전체 팀 투자 현황 테이블: 팀명, 총 투자금, 투자자 수, 현재 순위 (실시간 자동 정렬)
  * [투자 종료] 버튼, 결과 공유용 전체 화면 모드

### 7. 예외 및 상태 처리

* 남은 투자금 0원 → 투자 버튼 비활성화
* 총 투자금 1,000만원 미사용 상태 종료 시 경고 표시
* 네트워크 오류 발생 시 재시도 안내
