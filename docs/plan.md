# 투자 프로그램 MVP 구현 계획

## 문제 정의
PRD에 정의된 요구사항 대비 현재 구현 상태를 분석하여 미구현 기능을 단계적으로 추가 구현한다.

## 현재 상태 분석

### ✅ 구현 완료
- [x] 투자자 이름 입력 및 사용자 생성
- [x] 1,000만원 자동 부여
- [x] 팀 리스트 실시간 조회
- [x] 팀 상세 페이지 기본 구조
- [x] 10만원 단위 투자 검증
- [x] 중복 투자 방지 (트랜잭션)
- [x] 남은 투자금 표시
- [x] 관리자 팀 추가/삭제
- [x] 전체 데이터 초기화
- [x] Firebase 실시간 동기화
- [x] 모바일 UI 스타일링
- [x] 본인 소속 팀 선택 

### ❌ 미구현 (Must-have)
- [ ] 투자 상태 뱃지 (미투자/투자 완료)
- [ ] 투자 금액 범위 제한 (10만~1,000만원)
- [ ] 투자 확인 모달
- [ ] 팀 설명 관리
- [ ] 팀별 PDF/이미지 업로드
- [ ] 투자 시작/종료 제어
- [ ] 관리자 대시보드 (순위, 투자자 수)
- [ ] 남은 투자금 0원 시 버튼 비활성화

### ❌ 미구현 (Nice-to-have)
- [ ] 전체 투자 진행률 바
- [ ] 팀별 투자 비율 시각화 (바 차트)

---

## 구현 계획 (단계별)

### Phase 1: 모바일 UI 기반 스타일링
> **목표**: 모바일 우선 반응형 UI 구현

- [ ] 1.1 공통 CSS 변수 및 기본 스타일 정의
- [ ] 1.2 index.html 모바일 레이아웃 적용
- [ ] 1.3 team.html 모바일 레이아웃 적용
- [ ] 1.4 admin.html 레이아웃 정리

**테스트**: 모바일 뷰포트(375px)에서 모든 페이지 정상 표시 확인

---

### Phase 2: 투자자 시작 화면 개선
> **목표**: 본인 소속 팀 선택 및 투자 시작 플로우

- [ ] 2.1 프로그램 타이틀 추가
- [ ] 2.2 본인 소속 팀 선택 드롭다운/리스트 추가
- [ ] 2.3 투자 시작하기 버튼으로 팀 리스트 진입
- [ ] 2.4 상단 고정 영역 (남은 투자금 표시)

**테스트**: 팀 선택 후 투자 시작 → 팀 리스트 화면 진입 확인

---

### Phase 3: 팀 리스트 화면 개선
> **목표**: 투자 상태 표시 및 진행률 시각화

- [ ] 3.1 팀 카드 UI 개선 (팀명, 설명, 상태 뱃지)
- [ ] 3.2 투자 완료/미투자 상태 뱃지 표시
- [ ] 3.3 전체 투자 진행률 바 추가
- [ ] 3.4 남은 투자금 0원 시 UI 피드백

**테스트**: 투자 후 해당 팀 카드에 "투자 완료" 뱃지 표시 확인

---

### Phase 4: 팀 상세 페이지 개선
> **목표**: 투자 금액 검증 강화 및 확인 모달

- [ ] 4.1 투자 금액 범위 제한 (10만~남은금액, 최대 1,000만원)
- [ ] 4.2 투자 확인 모달 구현 (팀명, 금액, 남은 금액)
- [ ] 4.3 이미 투자한 팀 접근 시 비활성화 처리
- [ ] 4.4 팀 설명 표시 영역 추가
- [ ] 4.5 PDF/이미지 뷰어 영역 (placeholder)

**테스트**: 투자 시 확인 모달 → 확인/취소 동작 검증

---

### Phase 5: 관리자 기능 강화
> **목표**: 팀 관리 및 투자 제어

- [ ] 5.1 팀 설명 입력/수정 기능
- [ ] 5.2 팀별 PDF/이미지 URL 입력 기능
- [ ] 5.3 투자 시작/종료 제어 버튼
- [ ] 5.4 투자 종료 시 투자자 페이지 비활성화

**테스트**: 투자 종료 상태에서 투자 버튼 비활성화 확인

---

### Phase 6: 관리자 대시보드
> **목표**: 실시간 투자 현황 및 순위 표시

- [ ] 6.1 전체 팀 투자 현황 테이블 (팀명, 총 투자금, 투자자 수, 순위)
- [ ] 6.2 실시간 자동 정렬 (투자금 기준)
- [ ] 6.3 팀별 투자 비율 바 차트
- [ ] 6.4 결과 공유용 전체 화면 모드

**테스트**: 투자 발생 시 순위 자동 업데이트 확인

---

### Phase 7: 예외 처리 및 마무리
> **목표**: 에러 핸들링 및 UX 개선

- [ ] 7.1 네트워크 오류 시 재시도 안내
- [ ] 7.2 총 투자금 미사용 상태 경고
- [ ] 7.3 뒤로가기 버튼 추가
- [ ] 7.4 전체 테스트 및 버그 수정

**테스트**: 전체 시나리오 E2E 테스트

---

## 기술 스택
- **Frontend**: HTML, CSS (Vanilla)
- **Backend**: Firebase Firestore (실시간 DB)
- **Storage**: Firebase Storage (PDF/이미지, Phase 5)

## 파일 구조
```
├── index.html      # 투자자 시작 + 팀 리스트
├── team.html       # 팀 상세 + 투자
├── admin.html      # 관리자 대시보드
├── style.css       # 공통 스타일
├── firebase.js     # Firebase 설정
└── docs/
    └── PRD.md      # 요구사항 문서
```

## 참고사항
- 각 Phase는 독립적으로 구현/테스트 가능
- Phase 1-4는 투자자 경험, Phase 5-6은 관리자 경험 중심
- Phase 7은 전체 통합 및 예외 처리
